"Labyrinth" (1986)

その迷路は壁はない、しかしそれぞれの端をやぶが取り囲んでいる。 もしプレイヤーがやぶへ移動したら彼らは迷う。 迷路は行列(リストのリスト)で表される、 1 がやぶで0が経路の部分である。 迷路のサイズは12 x 12で外側の部屋もやぶである。 プレイヤーは(1,1)の部屋にいる。出口は(10, 10)の部屋である。 あなたは迷路を通り抜ける道筋を探す必要がある。 プレイヤーは4つの方向にしか動けない ー 南 (下 [1, 0])、北 (上 [-1, 0])、東(右 [0, 1])、西(左 [0, -1])。 道筋は次からなる文字列で表される。"S" が南、"N"が北、"E"が東、"W"が西。

入力 1と0からなるリストのリストで表された迷路のマップ

出力 "W" "E" "N" "S" を含む道筋を表す文字列

どうやって使われるか これは古典的なグラフの経路探索問題である ― もちろん、迷路はグラフとして表すことができる。 それはaからbへと導くナビゲーションソフトに使われており、コンピューターゲームの人工知能にも使われる。 あなたは望みどおりに道を探すことができる、マップを四角の部屋に分割して「悪い」部屋として印を付けるように。
